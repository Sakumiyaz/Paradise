# Paradise v0.2.0 Public Readiness

Status: prepared public readiness release. GPU training, production checkpoint
admission and AGI claims remain out of scope.

## Included

- Native Paradise/GARM/GEWC runtime readiness gates.
- Public contract validation for schemas and OpenAPI documents.
- Public checkpoint registry smoke with checkpoint admission blocked.
- Release package manifest under `target/paradise_release/`.
- Strict scanner path for `gitleaks` and `trufflehog`.
- Public demo transcript generation.
- Dataset manifest and module-level semantic coverage evaluation.
- Checkpoint evidence review that can inspect local probe reports without
  admitting weights.

## Verification

```bash
make contracts-validate
make paradise-non-gpu-readiness
make paradise-dataset-manifest
make paradise-module-semantic-eval
make paradise-checkpoint-evidence-review
make paradise-public-demo
make paradise-release-package
make public-audit
make check
make eden-api-conformance
```

Optional strict scanner gate:

```bash
make install-secret-scanners
make public-audit-strict
```

## Not Included

- No checkpoint weights.
- No production inference release.
- No GPU training/evaluation run.
- No external AGI benchmark claim.
- No autonomous high-risk action authority.

## GitHub Release Note

Use this note as the body for the GitHub release attached to the generated
`paradise-public-readiness-*` tag. The release should attach only public,
non-confidential evidence such as validation reports, demo transcripts and
release package manifests. Do not attach checkpoints, private datasets, `.env`
files, GPU VM workspaces or raw secrets scans.
