# Paradise v0.1.0-rc1

Date: 2026-05-24

Status: local release candidate. This is not an AGI capability claim and not a
trained model release.

Last local RC gate run: `make eden-release-check` on 2026-05-24.

## Scope

This release candidate is for the public local-first Paradise runtime
surface:

- GARM/GEWC operator runtime;
- runtime and operational APIs;
- `edenctl` operator CLI;
- `eden_core::sdk` Rust client;
- `contracts/v1` public contract directory;
- local conformance, black-box and long-run validation gates;
- native Locus/Forge command APIs and the governed Locus/Forge-to-CWM
  hypothesis bridge;
- operator console and release evidence path.

## Required Local Gates

Canonical full gate:

```sh
make eden-release-check
```

Expanded gate set:

```sh
make fmt
make check
make test
make security
make smoke
make public-audit
make eden-api-contracts
make eden-api-conformance
make operational-blackbox
make long-run-stability
```

The RC gate must keep `claim_allowed=false` and `agi_claim=false` in generated
validation, package and conformance artifacts.

## Claim Boundary

The RC validates local runtime behavior, API contracts, governance markers and
evidence packaging. It does not validate AGI, benchmark superiority, trained
LLM/LMM capability or autonomous deployment readiness.

## Operator Notes

Release gates start from clean temporary state and set
`EDEN_GARM_SKIP_LEGACY_MIGRATION=1` so stale local files under `/tmp/eden_garm_*`
cannot contaminate reproducibility evidence.

Optional local API protection is available by setting `EDEN_API_TOKEN` before
starting the runtime and sending either:

```text
Authorization: Bearer <token>
X-EDEN-API-Token: <token>
```

Public local routes for console, help, health and readiness remain readable so
operators can inspect startup state.
