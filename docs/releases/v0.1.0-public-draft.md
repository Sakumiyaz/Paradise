# v0.1.0-public-draft

Status: draft notes only. No versioned GitHub release has been published.

## Summary

This draft checkpoint documents the public Paradise source state. It
emphasizes operator clarity, local API conformance, claim boundaries and secret
hygiene.

## Highlights

- GEWC-governed GARM runtime remains the main operational surface.
- `docs/EDEN_SYSTEM_LAYERS.md` clarifies that Paradise is the public
  operator/runtime surface for a broader Eden architecture, not the completed
  Eden system.
- Static EDEN operator console is served from the runtime root endpoint.
- Runtime state, artifact, operational, capability, GEWC, validation and action
  contract APIs are documented.
- Rust SDK and live-endpoint conformance suite are available.
- Public-ready documents were added: license, security policy, contribution
  guide, changelog, code of conduct, claim limitations and threat model.
- Generated reports, backups, debug artifacts and Python bytecode were removed
  from the current tracked tree.

## Validation

Recommended before a versioned release:

```bash
make public-audit
make fmt
make check
make eden-api-conformance
make eden-api-contracts
```

## Known Limitations

- This is not an AGI claim.
- Local LMM/foundation-model training is not complete.
- Validation is local engineering evidence, not external certification.
- The GARM HTTP API is a localhost operator surface and should not be exposed
  directly to the internet.
- History rewrite has not been performed; see
  `docs/HISTORY_REWRITE_PLAYBOOK.md`.

## Release Decision

Do not publish this as a versioned GitHub release until the owner explicitly
approves the release notes and claim boundary.
