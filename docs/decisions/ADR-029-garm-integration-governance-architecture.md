# ADR-029: Formalize the GARM Integration Governance Architecture

## Status
Accepted

## Date
2026-05-22

## Context
EDEN/GARM now has formal architecture, frontier and paradigm artifacts. Those artifacts avoid duplicate taxonomy, but they still need a first-class integration layer so the system is not only a collection of disconnected modules.

The required integration surface includes central executive control, safe continual learning, physical/social grounding, causal world modeling, robust metacognition, stable/correctable objectives, complete AGI evaluation, scalable alignment and action governance boundaries.

## Decision
Add `integration governance eval`.

The command writes `integration_governance_architecture.json` with ten local no-claim cases:

- `module_integration_mechanism`
- `central_executive_control`
- `safe_continual_learning`
- `physical_social_grounding`
- `causal_world_model`
- `robust_metacognition`
- `stable_correctable_goals`
- `complete_agi_evaluation`
- `scalable_alignment`
- `action_governance_boundaries`

The artifact is included in the readiness package, capability registry and independent package validator.

## Consequences
EDEN can validate that its formal paradigms and frontier layers are coordinated by an executive, governed and corrigible integration architecture. The artifact remains local evidence only and preserves `claim_allowed=false` and `agi_claim=false`.
