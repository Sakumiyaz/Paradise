# ADR-057: Operational Contract And Black-Box Runtime Gates

## Status
Accepted

## Date
2026-05-24

## Context
The runtime now has two different verification needs:

- sandbox-safe tests that exercise routing and command logic without opening
  local sockets;
- real-process checks that prove the public runtime can start, serve TCP
  endpoints, expose contracts, execute governed local commands and shut down.

Keeping these in one test path caused environment-specific socket failures and
made it unclear whether a failure was in runtime logic or transport setup.

## Decision
Separate the verification layers:

- `make test` remains the default sandbox-safe Rust test gate.
- `make api-socket-test` explicitly validates TCP transport.
- `make operational-blackbox` launches EDEN as an external local process and
  validates health, readiness, operational contracts, dry-run hardening,
  runtime phase evidence, scenario evidence, replay and action evidence.
- `/api/operational/contract` serves the executable health/readiness/degraded
  mode/action-boundary contract generated by `operational api eval`.

GitHub Actions now treats socket transport and operational black-box validation
as separate jobs from sandbox-safe Rust tests.

## Alternatives Considered

### Keep socket tests inside the default Rust suite

This catches transport regressions early, but it makes default tests dependent
on the host's network namespace and fails in restricted sandboxes.

### Only document the operational contract

Documentation is useful, but without an endpoint and black-box gate it does not
prove the runtime exposes the contract from the same surface external consumers
use.

### Reuse the large API smoke script only

The existing smoke test covers many commands, but it is broad. The black-box
gate is narrower and focused on the operational runtime contract.

## Consequences

- Test failures are easier to classify: logic, TCP transport or black-box
  runtime behavior.
- External consumers can inspect `/api/operational/contract` directly.
- CI has more jobs, but each job has a clearer responsibility.
- The contract remains a local engineering boundary and preserves
  `claim_allowed=false` and `agi_claim=false`.
