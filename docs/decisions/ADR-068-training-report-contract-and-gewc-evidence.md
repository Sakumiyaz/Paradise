# ADR-068: Training Report Contract And GEWC Evidence Admission

## Status

Accepted

## Context

EDEN needs a training path before the LMM side is trained, but that path must
not imply production model capability or AGI validation. The previous smoke
benchmark produced a JSON report, but the report was not governed by a formal
contract and was not admitted into GARM/GEWC as a runtime artifact.

## Decision

Add a versioned `eden.training.capability_report.v1` schema, a stdlib-only
validator, and a Markdown evidence report generated by `make training-smoke`.
Add `training evidence eval` as a GEWC-routed command that reads the smoke
report, rejects claim escalation, and writes
`training_capability_evidence.json` under the active GARM state directory.

The first trainable module remains a subordinate memory-retrieval baseline:
`eden-memory-retrieval-baseline`. It may produce retrieval hypotheses for the
benchmark, but it cannot write runtime memory, objectives or tool actions
directly.

## Consequences

- CI now produces both machine-readable and human-readable training evidence.
- The artifact API can expose training reports and the GEWC-admitted evidence.
- Release packages can include training evidence when `make training-smoke`
  runs before local validation.
- The repo still makes no external-validation or AGI claim.

## Non-Goals

- Train a production LLM/LMM.
- Commit model checkpoints.
- Treat smoke success as external validation.
- Let model output mutate GEWC state without verification.
