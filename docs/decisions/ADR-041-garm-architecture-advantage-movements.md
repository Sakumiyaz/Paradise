# ADR-041: GARM Architecture Advantage Movements

## Status

Accepted

## Context

EDEN's architecture now competes less with single model providers and more with agentic and AGI architecture frameworks: LangGraph, AutoGen/Magentic-One, OpenHands and OpenCog Hyperon. Adding more labels would weaken locality. The next step needs executable architecture contracts that make EDEN's differentiator measurable: a governed cognitive operating system centered on GEWC.

## Decision

Add `architecture advantage eval` as the single validation command for six architecture movements:

- `gewc_trace_spec.json`
- `capability_reality_matrix_v2.json`
- `cognitive_task_suite.json`
- `eden_agent_sdk_contract.json`
- `model_adapter_layer.json`
- `reproducible_demos.json`
- `architecture_advantage_eval.json`

The command keeps `claim_allowed=false` and treats the result as local architecture evidence, not external capability certification.

## Consequences

- EDEN now has explicit contracts for GEWC decision traces, capability metrics, cognitive task families, agent SDK permissions, model adapters and reproducible demos.
- Release-candidate packaging and independent validation require these artifacts.
- The local held-out harness includes an architecture advantage case.
- This improves EDEN's architecture position without pretending the LMM is trained or claiming AGI.
