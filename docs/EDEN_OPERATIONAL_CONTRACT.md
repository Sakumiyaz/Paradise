# EDEN Operational Contract

Date: 2026-05-24

This document defines the current local runtime contract for Paradise. It is
an engineering contract for the public GARM/GEWC surface, not a claim of AGI
capability.

The executable form of this contract is generated by `operational api eval` as
`operational_contract.json` and is served from:

```text
/api/operational/contract
```

The live status form is served from:

```text
/api/operational/status
```

## Runtime States

| State | Meaning | Contract |
| --- | --- | --- |
| Health | The process accepts local HTTP requests. | `/api/health` returns service status. Health does not imply readiness, autonomy or capability claims. |
| Readiness | Startup completed and synchronous commands may be accepted. | `/ready` must return `ready`; `/api/command_sync` rejects commands until readiness is true. |
| Degraded | The runtime remains inspectable while a capability, handler or resource is paused, isolated, disabled or not ready. | Read APIs, dry-runs, state inspection and lifecycle recovery remain allowed; high-risk action remains blocked. |
| Shutdown | The runtime receives an explicit queued shutdown request. | `quit` is submitted through the command route and process cleanup is owned by the caller/runtime. |

## Persistent Permission Boundary

`operational api eval` writes `operational_permissions.json` and the runtime
serves the same policy at:

```text
/api/operational/permissions
```

GEWC uses this file as a persistent action boundary. Local read, inspection and
governed local runtime actions are allowed. Remote network action is denied by
default unless the runtime was started with the explicit remote-crawl override.
Dry-runs expose the relevant persistent permission key before any command is
queued.

## Replay And Evidence

The runtime exposes a read-only replay index at:

```text
/api/operational/replay
```

Specific decisions can be inspected with:

```text
/api/operational/replay?decision_id=<id>
```

Replay is evidence-only. It reconstructs the recorded GEWC decision and
completion records; it does not re-execute tools, commands or external actions.
The black-box gate also writes `operational_evidence_bundle.json`, a compact
manifest of operational artifacts, logs and checksums for release review.

## Operational Management

Permission changes are command-routed, not exposed as public write APIs:

```text
operational permissions audit
operational permissions diff
operational permissions history
operational permissions set <permission> <allow|deny>
operational permissions restore
```

Recovery is also command-routed:

```text
operational recovery audit
operational recovery run
```

The read-only recovery plan is served at `/api/operational/recovery`. The
runtime also exposes `/api/operational/schemas` as the local schema registry and
`/api/operational/demos` for reproducible operational demo evidence.

## Required Invariants

- GEWC remains the authority for command routing and action gating.
- Read APIs must not mutate runtime state.
- Dry-run endpoints must not queue or execute commands.
- Mutating commands must pass through the command queue and GEWC body registry.
- Model outputs are proposals until accepted by GEWC.
- Memory writes require an operational transaction or command-routed memory
  handler.
- High-risk or external actions require explicit capability, sandboxing and
  supervision.
- Generated evidence preserves `claim_allowed=false` and `agi_claim=false`.

## Verification

Use these gates for the operational contract:

```bash
make test
make api-socket-test
make operational-blackbox
make eden-api-conformance
```

`make test` is sandbox-safe and does not open local sockets. The TCP transport
and full black-box runtime gates are intentionally separate because they launch
a local process and validate the API from outside the runtime.
