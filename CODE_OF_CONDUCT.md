# Code Of Conduct

## Our Standards

Participants are expected to keep technical discussions precise, respectful and
focused on improving the project.

Examples of encouraged behavior:

- discussing risks and limitations clearly;
- separating evidence from claims;
- respecting safety boundaries;
- giving actionable feedback.

Examples of unacceptable behavior:

- harassment or personal attacks;
- publishing private information;
- posting credentials, tokens or private runtime artifacts;
- encouraging unsafe deployment of command-capable endpoints.

## Enforcement

Report concerns privately to the repository maintainers. Maintainers may remove
content, restrict participation or reject contributions that violate these
standards.

This project is experimental. Safety and privacy concerns take priority over
feature velocity.
